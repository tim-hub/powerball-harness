# X記事向け画像生成

このファイルは、X 記事で使う **ヘッダー画像** と **差し込み画像** を作るときの進め方です。

この repo では、`claude-code-harness` の記事画像は次の前提で作る。

- **Nano Banana Pro（Gemini 画像生成 API）を使う**
- **白背景メインのミニマルなデザインにする**
- **`docs/images/claude-harness-logo-with-text.png` を参照画像として渡す**
- **画像内テキストは日本語を基本にする**

## 1. 基本方針

- ヘッダー画像は「開かせる」ための画像
- 差し込み画像は「理解させる」ための画像
- 1 記事につき、最低でも次を用意する
  - ヘッダー画像 1 枚
  - 差し込み画像 2 枚

## 2. 使うモデル

Nano Banana Pro として扱っている、Gemini の画像生成 API を使う。

### API 仕様

| 項目 | 値 |
|------|-----|
| モデル | `gemini-3-pro-image-preview` |
| エンドポイント | `https://generativelanguage.googleapis.com/v1beta/models/gemini-3-pro-image-preview:generateContent` |
| 認証 | `x-goog-api-key: ${GOOGLE_AI_API_KEY}` または `?key=${GEMINI_API_KEY}` |
| 主な出力 | `candidates[0].content.parts[].inlineData.data` |

> 補足: 認証変数はこの repo では `GOOGLE_AI_API_KEY` を優先する。既存の `GEMINI_API_KEY` がある場合はフォールバックとして扱ってよい。

## 3. 画像の役割分担

### ヘッダー画像

目的:
- 記事テーマをひと目で伝える
- タイトルと一緒に見たときの引力を上げる

入れるもの:
- 主題を象徴する 1 つの強いビジュアル
- 必要なら短い補助テキスト
- 余白

避けるもの:
- 小さすぎる文字
- 情報の詰め込み
- 記事タイトルの全文コピペ
- 暗色ベタ塗りの generic AI バナー
- ロゴの後付け感が強い構図

### 差し込み画像

使いやすい型:

1. **比較型**
   - Before / After
2. **整理型**
   - 3〜5 要点のカード
3. **流れ型**
   - ステップ図
4. **引用型**
   - 強い一文を視覚化

## 4. ブランドトーン

### 必須

- 背景は白を主役にする
- 情報量を絞り、余白を広く取る
- `claude-code-harness` ロゴを適切に使う
- 画像内の主要テキストは日本語にする

### アクセント

- オレンジ: ロゴや強調の補助
- 深い青: 見出しや骨格
- 青緑 / スレート: 補助要素

### 避けること

- 紫一色の AI バナー
- グラデーション過多
- ロゴの再描画
- 英語主体の見出し
- 文字量の多いポスター風デザイン

## 5. Prompt の作り方

### 共通ルール

- 何の記事かを最初に書く
- 誰向けかを入れる
- 雰囲気ではなく、構図と役割を書く
- 文字を入れるなら短くする
- 画像の安全域を広く取るよう指示する
- 白背景メイン、ミニマル、日本語主体を毎回明示する
- ロゴ参照画像を提供し、ロゴを新しく描かないよう指示する

### ヘッダー画像テンプレート

```text
X記事のヘッダー画像を生成してください。

テーマ: {THEME}
読者: {AUDIENCE}
記事の約束: {PROMISE}

目的:
- 記事を開きたくなる第一印象を作る
- テーマがひと目で伝わる

デザイン要件:
- 横長で、中央に重要要素を置く
- 背景は白を主役にする
- 余白を広めに取る
- 情報は絞る
- 上品でプロフェッショナル
- ノイズを減らす
- 画像内の主要テキストは日本語
- 提供した claude-code-harness ロゴを自然に使う
- ロゴは再描画せず、参照画像の印象を保つ

避けること:
- 小さい文字の多用
- テキストの重複
- ありがちな generic SaaS バナー
- 黒背景のヒーロービジュアル
```

### 差し込み画像テンプレート

```text
X記事の差し込み画像を生成してください。

テーマ: {THEME}
対象セクション: {SECTION_TITLE}
伝えたい要点: {KEY_POINT}

画像の役割:
- このセクションの理解を助ける
- 長文の流れを切って読みやすくする

希望する表現:
- {COMPARISON | FLOW | CARD | QUOTE}
- 情報は 1 テーマに絞る
- 一目で意味が伝わる
- 文字を入れる場合は短く読みやすく
- 背景は白主体
- 日本語の短い見出しを使う
- 提供した claude-code-harness ロゴを小さく自然に入れる
```

## 6. ロゴ参照画像の使い方

ロゴ素材は次を使う。

```text
docs/images/claude-harness-logo-with-text.png
```

Gemini へは、テキストだけでなくロゴ画像も `inlineData` として渡す。

### 参考 JSON

```json
{
  "contents": [{
    "parts": [
      {
        "inlineData": {
          "mimeType": "image/png",
          "data": "<base64-logo>"
        }
      },
      {
        "text": "<prompt>"
      }
    ]
  }],
  "generationConfig": {
    "responseModalities": ["TEXT", "IMAGE"],
    "imageConfig": {
      "aspectRatio": "16:9",
      "imageSize": "2K"
    }
  }
}
```

この repo では、`scripts/generate-x-article-image.sh` を使うと、この組み立てを毎回やり直さずに済む。

## 7. 生成の進め方

### Step 1: 候補を出す

- ヘッダー画像は最低 3 候補
- 差し込み画像は各 1〜2 候補

### Step 2: 画面で確認する

次を必ず見る。

- テーマとズレていないか
- 文字が読めるか
- 情報が多すぎないか
- AI っぽい崩れがないか
- 白背景メインのブランドトーンを守れているか
- ロゴが再描画されず、自然に統合されているか
- 日本語テキストが崩れていないか

### Step 3: NG なら再生成する

よくある直し方:

- 情報が多い → 「要素を減らす」「1 つの主題だけに絞る」
- 文字が弱い → 「テキストは 1 フレーズだけ」「文字を大きく」
- 構図が散る → 「中央に主役を 1 つ」「余白を広く」
- 記事とズレる → テーマ、読者、要点を書き直す

## 8. 保存ルール

- ヘッダー画像: `out/social/<slug>/header/`
- 差し込み画像: `out/social/<slug>/inline/`
- プロンプトと alt テキスト: `out/social/<slug>/image-prompts.md`
- API リクエスト / レスポンス: `out/social/<slug>/responses/`

## 9. alt テキストの書き方

alt テキストは短すぎても長すぎてもよくない。次の順で書く。

1. 何の画像か
2. どんな要素が見えるか
3. 記事のどの主張に対応しているか

例:

- `Claude Code の更新点を Harness の運用フローに接続していることを示す図。中央の矢印が、単体機能から運用改善への変換を表している。`

## 10. 注意

- X 公式ヘルプは「ヘッダー画像が重要」とは述べているが、この記事執筆時点でこのヘルプページには厳密なサイズ指定は見当たらない
- そのため、サイズは **公式仕様として断定せず**、「横長・中央安全域・余白広め」を優先する
- API キーは repo 直下の `.env` かシェル環境に置き、本文やログに値を出さない
