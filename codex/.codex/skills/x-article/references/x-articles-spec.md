# X記事の公式仕様メモ

このメモは、X 公式ヘルプ「記事機能について」を、実務で使いやすい形に整理したものです。

公式ページ:
- https://help.x.com/ja/using-x/articles

## 1. X公式が明言していること

### 利用条件

- X 公式ヘルプでは、記事機能は **プレミアム会員、プレミアムプラス会員、プレミアムビジネス会員、プレミアム組織会員** 向けと案内されている

### 記事に入れられるもの

- テキスト
- 画像
- ビデオ
- GIF
- ポスト
- リンク

### 書式

- 見出し
- 小見出し
- 太字
- 斜体
- 取り消し線
- インデント
- 番号付きリスト
- 箇条書きリスト

### 基本操作

- `記事` タブから作成する
- 公開済み記事は編集時にいったん非公開になり、編集後に再公開できる
- 記事は削除できる

## 2. X公式の「良い記事」の考え方

X 公式ヘルプの主張を要約すると、次の 8 点です。

1. **明確な目的から始める**
   - 読後に読者へ何を思ってほしいか、感じてほしいか、してほしいかを決める
   - 誰向けの記事かを明確にする
2. **タイトルと冒頭フックを強くする**
   - タイトルは具体的で、好奇心を刺激し、価値を約束するものがよい
   - 冒頭に力強い一文を置く
   - ヘッダー画像も重要
3. **流し読みしやすい構造にする**
   - 短い段落
   - 小見出し
   - 箇条書き
   - 太字
   - 1段落1アイデア
4. **自然で理解しやすい文体にする**
   - 会話に近い文体を保つ
5. **主張だけでなく、実例や証拠を示す**
   - データ、体験談、ビフォーアフター、埋め込み投稿など
6. **容赦なく編集する**
   - 単語数を 20〜30% 削る
   - 冗長表現を減らす
7. **ビジュアルと書式を加える**
   - 画像やスクリーンショットで区切る
   - 共有されやすくする
8. **締めを強くする**
   - 要点をまとめる
   - 質問や行動の促しを入れる

## 3. X公式の「伸ばし方」

X 公式ヘルプでは、公開後の伸ばし方として次を案内している。

- 公開前に予告する
- 新しい記事をプロフィールにピン留めする
- 記事の重要部分をポストのスレッドに分けて共有する
- 早期読者への返信を積極的に行う
- 関連性があれば古い記事も再共有する

## 4. Harnessでの実務解釈

ここから下は **公式仕様そのものではなく、Harness で記事を強くするための運用解釈** です。

### 記事パッケージに最低限入れるもの

- タイトル候補 3 本以上
- 冒頭フック
- 小見出し付き本文
- ヘッダー画像
- 差し込み画像 2〜4 枚
- alt テキスト
- 公開後の予告ポスト案と抜粋ポスト案

### 画像の役割分担

- ヘッダー画像:
  - 開封率を上げる入口
  - 記事テーマをひと目で伝える
- 差し込み画像:
  - 理解を助ける
  - 長文の疲れを減らす
  - 引用・共有しやすい要点を可視化する

### 記事で狙うべき体験

- 「長い」ではなく「止まらず読める」
- 「きれい」ではなく「要点が頭に残る」
- 「説明した」ではなく「読み手が動ける」

## 5. この skill で守ること

- 公式仕様として確認できることは、なるべく一次情報に寄せる
- 文字数や画像サイズなど、公式ヘルプに明記がないものは **Harness推奨** として扱う
- 公式の推奨と矛盾する記事構成にしない
