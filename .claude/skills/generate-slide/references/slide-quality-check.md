# Slide Quality Check - Slide Image Quality Assessment

Claude visually evaluates slide images generated by Nano Banana Pro to ensure quality.

---

## Overview

This is the quality assessment logic executed in Step 4 of `/generate-slide`.
It evaluates 2 images for each pattern (Minimalist, Infographic, Hero Visual) and selects the best one.

---

## Assessment Flow

```
Receive 2 images per pattern
    |
    +--[Step 1] Evaluate each image individually
    |   +-- Score using 5 slide-specific criteria
    |   +-- Rate each criterion on a 1-5 scale
    |   +-- Calculate overall score (weighted average)
    |
    +--[Step 2] Compare within pattern
    |   +-- Both score 3+ → adopt the higher one
    |   +-- Only one scores 3+ → adopt that one
    |   +-- Both score 2 or below → regenerate
    |
    +--[Step 3] Regeneration decision
        +-- Both NG → improve prompt and regenerate (max 3 times)
        +-- Retry limit reached → report to user
```

---

## Slide-Specific Assessment Criteria

### 5 Criteria and Weights

| Criterion | Weight | Description | Evaluation Points |
|-----------|--------|-------------|-------------------|
| **Information Delivery** | High (x3) | Can the project's features be conveyed in a single slide? | Project purpose instantly understandable, value proposition clear |
| **Layout Balance** | High (x3) | Visual completeness, use of whitespace | Elements well-organized, natural visual flow |
| **Text Readability** | Medium (x2) | Is AI-generated text readable? | No garbled text, appropriate font size, sufficient contrast |
| **Professionalism** | Medium (x2) | Quality suitable for business use? | Not cheap-looking, refined color usage, consistent design |
| **Brand Consistency** | Low (x1) | Match with specified colors/tone | Aligned with specified tone, colors not significantly off |

### Overall Score Calculation

```
Overall Score = (Information Delivery x 3 + Layout x 3 + Text Readability x 2 + Professionalism x 2 + Brand Consistency x 1) / 11
```

### Score Definitions

| Score | Verdict | Description |
|-------|---------|-------------|
| 5 | Excellent | Perfect, adopt immediately |
| 4 | Good | Good quality, acceptable |
| 3 | Acceptable | Within tolerance, adopt if no better option |
| 2 | Poor | Has issues, regeneration recommended |
| 1 | Unacceptable | Unusable, must regenerate |

### Adoption Threshold

```
Adoption threshold = 3 (Acceptable or above)
```

## Pattern-Specific Additional Checks

### Minimalist

| Check Item | Pass Criteria |
|------------|---------------|
| Whitespace usage | Sufficient whitespace, not cramped |
| Typography | Fonts readable, clear hierarchy |
| Simplicity | Elements kept to a minimum |

### Infographic

| Check Item | Pass Criteria |
|------------|---------------|
| Information structure | Visually segmented into sections |
| Icon/diagram usage | Visual elements present, not just text |
| Data visualization | Numbers and features illustrated |

### Hero Visual

| Check Item | Pass Criteria |
|------------|---------------|
| Visual impact | Eye-catching bold visuals |
| Catchcopy | Message clear and readable |
| Emotional appeal | Conveys project value on a sensory level |

## Pattern Selection Logic

### Basic Rules

```
1. Both score 3 or above:
   → Adopt the higher score
   → Same score → determine by priority criteria:
      1. Higher information delivery
      2. Higher professionalism
      3. Default to candidate 1

2. Only one scores 3 or above:
   → Adopt the one scoring 3+

3. Both score 2 or below:
   → Regenerate (improve prompt)
   → If retry limit (3 times) reached:
      → Provisionally adopt the higher score and report to user
      → User chooses to continue or skip
```

### Retry Control

```
max_retries = 3 (per pattern)
```

| Retry | Improvement Strategy |
|-------|---------------------|
| 1st | Generate with initial prompt |
| 2nd | Reflect quality feedback. Add specific modifiers |
| 3rd | Significantly change style. Make composition instructions more specific |
