# X投稿作成 - 新規作成

## 概要

Claude harness プロモーション用のX投稿を新規作成します。

---

## Step 1: ターゲット確認

ユーザーに確認:

「どのターゲット向けの投稿ですか？」

| ターゲット | 特徴 | キーメッセージ |
|-----------|------|---------------|
| エンジニア | 技術的な話が刺さる | 品質保証、3層防御、テスト改ざん防止 |
| バイブコーダー | 技術用語を避ける | 日本語で話すだけ、プログラミング不要 |
| 興味ある人 | ハードルの低さを強調 | 5分で始められる、3コマンドだけ |
| 再チャレンジ | 改善点を強調 | 以前より簡単、Skills Gate、Memory |
| 汎用 | 誰にでも刺さる | Plan→Work→Review、自律運用 |

---

## Step 2: 投稿タイプ選択

| タイプ | 特徴 | 適した場面 |
|--------|------|-----------|
| 問題提起型 | 「〇〇な経験ありませんか？」 | 共感を得たいとき |
| Before/After型 | 変化を明示 | 価値を伝えたいとき |
| Tips型 | 具体的なノウハウ | 既存ユーザー向け |
| アップデート告知型 | 新機能紹介 | リリース時 |
| 引用リポスト型 | 他の投稿に便乗 | バズ狙い |

---

## Step 3: 投稿文作成

### 必須要素チェックリスト

- [ ] **1行目がフック** になっているか（読者の注意を引く）
- [ ] **プロフィールキーワード** を含んでいるか
- [ ] **具体的な価値** を述べているか
- [ ] **行動喚起** があるか（試す、見る、クリック）
- [ ] **ハッシュタグ** は2-3個か

### 文字数ガイド

| 要素 | 推奨文字数 |
|------|-----------|
| 本文 | 100-200文字 |
| ハッシュタグ | 20-40文字 |
| 合計 | 140-240文字 |

280文字ギリギリは避ける（読みづらい）

---

## Step 4: 画像・動画の検討

### 優先度

1. **スライド画像** - NotebookLM で生成したもの
2. **スクリーンショット** - 実際の動作画面
3. **Before/After 画像** - 比較がわかるもの
4. **コード例** - シンタックスハイライト付き

### 推奨

- 4枚まで添付可能
- 1枚目は最もインパクトのあるもの
- 引用リポストの場合は必ず画像を添付

---

## Step 5: 最終チェック

### 投稿前チェックリスト

- [ ] プロフィールキーワードを含んでいる
- [ ] 1行目でフックできている
- [ ] 画像または動画を添付した
- [ ] ハッシュタグは適切か
- [ ] URLを貼る場合、短縮せず貼った
- [ ] 10分以内にいいねがつかなければ削除する覚悟がある

### NG チェック

- [ ] プロフィールにないカテゴリを扱っていないか
- [ ] 専門用語が多すぎないか（ターゲット次第）
- [ ] ネガティブすぎないか
- [ ] 長すぎないか

---

## 出力フォーマット

```
【投稿案】

[本文]

#ハッシュタグ1 #ハッシュタグ2

---

【添付推奨】
- [画像/動画の説明]

【投稿タイミング】
- [推奨時間帯]

【備考】
- [その他のアドバイス]
```

---

## 例: エンジニア向け問題提起型

```
【投稿案】

AIがテスト失敗を「テスト修正」で解決しようとする問題、解決しました。

Claude harness の3層防御戦略:
📜 Rules - 常時適用のガイドライン
🎯 Skills - 状況に応じた自動発動
🔒 Hooks - 技術的に改ざんをブロック

もうテストを壊されない。

🔗 github.com/Chachamaru127/claude-code-harness

#ClaudeCode #AI開発 #品質保証

---

【添付推奨】
- 3層防御の図解スライド
- 実際のテスト改ざんコード例のスクショ

【投稿タイミング】
- 平日 19:00-21:00（帰宅後のエンジニアに刺さる）

【備考】
- 問題提起から入るので共感を得やすい
- 具体的な解決策を示しているので行動につながりやすい
```
