# X投稿作成 - 最適化

## 概要

既存の投稿案をXアルゴリズム最適化のノウハウに基づいて改善します。

---

## Step 1: 投稿案を受け取る

ユーザーから投稿案を受け取り、以下の観点で分析:

| 観点 | チェック内容 |
|------|-------------|
| フック | 1行目で注意を引けるか |
| 明確さ | 何を伝えたいか明確か |
| 価値 | 読者にとってのメリットがあるか |
| 行動喚起 | 次のアクションを促しているか |
| キーワード | プロフィールキーワードを含むか |

---

## Step 2: 問題点を特定

### よくある問題

#### A. フックが弱い

**Before**: 「Claude harness を紹介します」
**After**: 「AIがテストを壊す問題、解決しました」

#### B. 抽象的すぎる

**Before**: 「便利なプラグインです」
**After**: 「5分で始められて、3つのコマンドだけ覚えればOK」

#### C. 価値が不明確

**Before**: 「Plan → Work → Review のサイクルで動きます」
**After**: 「計画→実装→レビューを自動化。もう手順に迷わない」

#### D. 長すぎる

280文字ギリギリは避ける。100-200文字が読みやすい。

#### E. ハッシュタグが多すぎる

2-3個で十分。5個以上はスパムっぽく見える。

---

## Step 3: 改善案を提示

### 改善フォーマット

```
【元の投稿】
[元の投稿文]

【問題点】
1. [問題1]
2. [問題2]

【改善案A】
[改善版1]

【改善案B】
[改善版2 - 別アプローチ]

【推奨】
[AまたはBを推奨する理由]
```

---

## Step 4: バズ要素の追加

### バズりやすい要素

| 要素 | 例 |
|------|-----|
| 数字 | 「5分」「3つ」「100%」 |
| 対比 | 「Before/After」「〇〇 vs △△」 |
| 驚き | 「まさかの」「実は」「知らなかった」 |
| 共感 | 「あるある」「わかる」 |
| 実績 | 「〇〇人が使用」「★4.8」 |

### 弱い → 強い 変換例

| 弱い | 強い |
|------|------|
| 便利です | 作業時間が半分に |
| 使いやすい | 5分で始められる |
| 品質が上がる | テスト改ざんをゼロに |
| おすすめ | これ一択 |

---

## Step 5: 引用リポスト用の最適化

引用リポストの場合、特別な最適化が必要:

### 必須ルール

1. **1行目で明確な賛成 or 反対**
2. **理由を補足**
3. **画像を必ず添付**（引用先の画像は消えやすい）

### 引用リポスト構成

```
[賛成/反対の表明]

[理由1]
[理由2]

[追加の視点または補足]

#ハッシュタグ
```

### 例

```
【元の投稿（引用先）】
「AI開発はまだ実用段階じゃない」

【改善案】
これは半分正しくて半分間違い。

確かに素のClaude Codeだと暴走することもある。
でも Claude harness を入れると：
・テスト改ざんを自動ブロック
・品質ガードレールが常時監視
・セッション跨ぎで学習

適切なハーネスがあれば、もう実用段階です。

#ClaudeCode #AI開発
```

---

## Step 6: 最終確認

### 最適化後チェックリスト

- [ ] 元の意図を保持しているか
- [ ] より短くなったか（または同等）
- [ ] より具体的になったか
- [ ] フックが強くなったか
- [ ] 行動喚起があるか

### A/Bテスト提案

可能であれば、2パターンの投稿案を提示:

| パターン | 特徴 |
|---------|------|
| A | 問題提起型・共感重視 |
| B | 価値訴求型・具体性重視 |

両方投稿して、反応が良い方を今後の参考にする。
