# Image Quality Check - Claude Image Quality Assessment

Claude visually evaluates images generated by Nano Banana Pro to ensure quality.

---

## Overview

This is the quality assessment logic executed in Step 4 of `image-generator.md`.
Claude analyzes 2 generated images and determines adoption/rejection.

## Assessment Flow

```
Receive 2 images
    |
    +-- [Step 1] Evaluate each image individually
    |   +-- Basic quality check
    |   +-- Scene fit check
    |   +-- Brand consistency check
    |
    +-- [Step 2] Aggregate results
    |   +-- Both OK → go to Step 3
    |   +-- One OK → adopt the OK one
    |   +-- Both NG → regenerate
    |
    +-- [Step 3] Select the best from 2 images
        +-- Comparative evaluation
        +-- Determine adopted image
```

---

## Assessment Criteria

### 1. Basic Quality Check

| Check Item | Pass Criteria | NG Examples |
|------------|---------------|-------------|
| **Sharpness** | No blur or noise | Out of focus, excessive noise |
| **Composition** | Well-balanced | Bias, cropping, unnatural placement |
| **Color tone** | Natural and consistent | Color unevenness, unnatural saturation |
| **Artifacts** | No generation noise | Distortion, repeating patterns |
| **Resolution** | Meets specified resolution | Low resolution, upscale degradation |

### 2. Scene Fit Check

| Check Item | Pass Criteria | NG Examples |
|------------|---------------|-------------|
| **Subject match** | Expresses the prompt's subject | Unrelated content |
| **Style match** | Follows specified style | Style mismatch |
| **Purpose fit** | Suits the scene's purpose | Composition unsuitable for video |
| **Text quality** | Readable if included | Garbled text, unnatural text |

### 3. Brand Consistency Check

| Check Item | Pass Criteria | NG Examples |
|------------|---------------|-------------|
| **Color scheme** | Harmonizes with brand colors | Color mismatch |
| **Tone** | Professional | Too casual, cheap-looking |
| **Consistency** | Unified feel with other scenes | Looks out of place |

---

## Evaluation Scoring

### Score Definitions

| Score | Verdict | Description |
|-------|---------|-------------|
| 5 | Excellent | Perfect, adopt immediately |
| 4 | Good | Good quality, acceptable |
| 3 | Acceptable | Within tolerance, adopt if no better option |
| 2 | Poor | Has issues, regeneration recommended |
| 1 | Unacceptable | Unusable, must regenerate |

### Adoption Threshold

```
Adoption threshold = 3 (Acceptable or above)
```

### Selection Logic When Comparing 2 Images

```
When both are above threshold:
  → Adopt the higher score
  → If same score, decide by:
     1. Higher scene fit
     2. Higher brand consistency
     3. Adopt image 1 (default)
```

### Image Evaluation Prompt

````text
Please evaluate the following image.

## Evaluation Target
- Scene: {scene_name}
- Purpose: {scene_purpose}
- Expected style: {expected_style}

## Evaluation Criteria
1. Basic quality (sharpness, composition, color tone, artifacts)
````

## Regeneration Instructions on NG

### Improvement Prompt Generation

```
The previous image was rejected.

## Issues
{issues}

## Improvement Suggestions
{improvement_suggestions}

## Revised Prompt
Please generate an improved prompt addressing the following:
1. {improvement_1}
2. {improvement_2}
```

### Adjustment Strategy by Problem Category

| Problem Category | Adjustment Strategy |
|-----------------|---------------------|
| Composition issues | Add `centered composition`, `balanced layout` |
| Color issues | Add specific color specifications (HEX) |
| Style mismatch | Make style specification more specific |
| Text quality | Add `no text`, overlay text later |
| Quality degradation | Add `high quality`, `4K`, `detailed` |

## Assessment Bypass

### Force Adoption

```
When user specifies "This image is fine":
→ Skip quality assessment and adopt
```

### Threshold Adjustment

```
When user specifies "Be stricter":
→ Raise adoption threshold to 4

When user specifies "Good enough for now":
→ Lower adoption threshold to 2
```
