# /webhook-setup - Webhook トリガーセットアップ

GitHub Actions から Harness コマンドを自動実行するための設定を行います。
**PR作成時に自動レビュー**などを実現できます。

## VibeCoder Quick Reference

- "**PRで自動レビューしたい**" → このコマンド
- "**CI でハーネス使いたい**" → Webhook トリガーをセットアップ
- "**自動化したい**" → GitHub Actions 連携

## Deliverables

- `.github/workflows/harness-review.yml` - PR自動レビューワークフロー
- `.github/workflows/harness-plan-check.yml` - Plans.md 整合性チェック（オプション）

---

## Usage

```bash
/webhook-setup
```

## Execution Flow

### Step 1: 機能選択

> 🔧 **どの自動化を設定しますか？**
>
> 1. PR自動レビュー（`/harness-review` をPR作成時に実行）
> 2. Plans.md 整合性チェック（PRにタスク完了状況をコメント）
> 3. 両方
>
> 番号で回答（デフォルト: 1）

**ユーザーの回答を待つ**

### Step 2: ワークフローファイル生成

選択に応じてワークフローファイルを生成：

#### PR自動レビュー（選択肢1, 3）

`.github/workflows/harness-review.yml` を生成：

```yaml
name: Harness Review

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    name: Auto Review
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Harness Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          # 変更ファイル一覧を取得
          CHANGED_FILES=$(git diff --name-only origin/${{ github.base_ref }}...HEAD)

          # レビュー実行（非対話モード）
          claude --non-interactive << 'EOF'
          /harness-review --ci --files "$CHANGED_FILES"

          レビュー結果を以下の形式でまとめてください：
          1. 重大な問題（あれば）
          2. 改善提案
          3. 良い点
          EOF

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');

            // レビュー結果を読み込み（存在する場合）
            let reviewBody = '## 🔍 Harness Auto Review\n\n';

            if (fs.existsSync('.claude/state/last-review.md')) {
              reviewBody += fs.readFileSync('.claude/state/last-review.md', 'utf8');
            } else {
              reviewBody += '✅ 自動レビューが完了しました。重大な問題は検出されませんでした。';
            }

            reviewBody += '\n\n---\n🤖 Generated by [Claude Code Harness](https://github.com/Chachamaru127/claude-code-harness)';

            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: reviewBody
            });
```

#### Plans.md 整合性チェック（選択肢2, 3）

`.github/workflows/harness-plan-check.yml` を生成：

```yaml
name: Plans Check

on:
  pull_request:
    types: [opened, synchronize]
    paths:
      - 'Plans.md'
      - 'src/**'

permissions:
  contents: read
  pull-requests: write

jobs:
  check:
    name: Plans Consistency
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Check Plans.md
        id: plans
        run: |
          if [ ! -f Plans.md ]; then
            echo "plans_exists=false" >> $GITHUB_OUTPUT
            exit 0
          fi

          echo "plans_exists=true" >> $GITHUB_OUTPUT

          # タスク状況をカウント
          WIP=$(grep -c "cc:WIP" Plans.md 2>/dev/null || echo "0")
          TODO=$(grep -c "cc:TODO" Plans.md 2>/dev/null || echo "0")
          DONE=$(grep -c "cc:DONE" Plans.md 2>/dev/null || echo "0")

          echo "wip=$WIP" >> $GITHUB_OUTPUT
          echo "todo=$TODO" >> $GITHUB_OUTPUT
          echo "done=$DONE" >> $GITHUB_OUTPUT

      - name: Post Status Comment
        if: steps.plans.outputs.plans_exists == 'true'
        uses: actions/github-script@v7
        with:
          script: |
            const wip = '${{ steps.plans.outputs.wip }}';
            const todo = '${{ steps.plans.outputs.todo }}';
            const done = '${{ steps.plans.outputs.done }}';

            const body = `## 📋 Plans.md Status

            | Status | Count |
            |--------|-------|
            | 🔄 WIP | ${wip} |
            | 📝 TODO | ${todo} |
            | ✅ DONE | ${done} |

            ---
            🤖 Generated by [Claude Code Harness](https://github.com/Chachamaru127/claude-code-harness)`;

            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: body
            });
```

### Step 3: シークレット設定ガイド

> ⚠️ **ANTHROPIC_API_KEY の設定が必要です**
>
> GitHub リポジトリで以下を設定してください：
>
> 1. Settings → Secrets and variables → Actions
> 2. "New repository secret" をクリック
> 3. Name: `ANTHROPIC_API_KEY`
> 4. Value: Anthropic API キーを入力
>
> 💡 API キーは https://console.anthropic.com で取得できます

### Step 4: 完了報告

> ✅ **Webhook トリガー設定完了**
>
> 📄 **生成ファイル**:
> - `.github/workflows/harness-review.yml` - PR自動レビュー
> - `.github/workflows/harness-plan-check.yml` - Plans.md チェック（選択時）
>
> **次のステップ:**
> 1. `ANTHROPIC_API_KEY` シークレットを設定
> 2. 変更をコミット＆プッシュ
> 3. PRを作成して動作確認
>
> 💡 ワークフローはPR作成時に自動実行されます

## Customization

### レビュー対象ファイルのフィルタリング

```yaml
on:
  pull_request:
    paths:
      - 'src/**'
      - '!src/**/*.test.ts'  # テストファイルは除外
```

### レビュー観点のカスタマイズ

```yaml
- name: Run Harness Review
  run: |
    claude --non-interactive << 'EOF'
    /harness-review --ci --focus security,performance
    EOF
```

### 特定ブランチのみ実行

```yaml
on:
  pull_request:
    branches:
      - main
      - develop
```

## Troubleshooting

### Error: ANTHROPIC_API_KEY not set

**原因**: シークレットが未設定

**解決**: リポジトリの Settings → Secrets で `ANTHROPIC_API_KEY` を設定

### Error: Permission denied

**原因**: ワークフローの権限不足

**解決**: `permissions` セクションを確認
```yaml
permissions:
  contents: read
  pull-requests: write
```

### レビューコメントが投稿されない

**原因**: `actions/github-script` の実行エラー

**解決**: Actions タブでログを確認
